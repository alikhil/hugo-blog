# AI Assistant Instructions

This directory contains documentation for AI coding assistants (Claude, GitHub Copilot, Cursor, etc.) to understand the repository structure and guidelines.

## Files

- **`.ai/REPOSITORY.md`** - Complete overview of repository structure, directories, configuration, and site features
- **`.ai/CONTENT_GUIDELINES.md`** - Writing style guidelines, post structure, and content best practices
- **`.ai/DEVELOPMENT.md`** - Development workflow, common tasks, and troubleshooting

## Purpose

These files help AI assistants:
- Understand the Hugo blog structure without exploring the repository
- Write content that matches the author's style and voice
- Follow consistent formatting and conventions
- Make appropriate technical decisions
- Provide accurate development guidance

## Usage

AI assistants should read these files when:
- Creating or editing blog posts
- Modifying site configuration
- Adding new features or customizations
- Answering questions about the repository
- Debugging issues

## Maintenance

Update these files when:
- Repository structure changes significantly
- New conventions or guidelines are established
- Hugo or theme versions are updated
- New features are added to the site

## Notes for AI Assistants

1. **Check existing posts** in `content/posts/` for style reference
2. **Follow CONTENT_GUIDELINES.md** for writing style (American English, simpler vocabulary)
3. **Use DEVELOPMENT.md** for technical workflows
4. **Reference REPOSITORY.md** for understanding directory structure
